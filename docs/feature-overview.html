Feature overview

ExcalidrawにAgentが参加し、提案をghostとして見せる

人間はいつも通りcanvasを編集する。Agentは同じYjs documentを読み、別ワークスペースで図を作り、 いきなり本体を書き換えずに「承認待ちの提案」としてcanvasへ表示する。

Note
「この構成をデプロイ図にして」
Agent ghost proposal 半透明・lockedの要素として表示。Approveするまで通常要素にはならない。

何ができるか

機能は「共同編集」「Agent実行」「レビューして反映」の3つに分けると見通しやすい。

Human canvas

ブラウザでExcalidrawを編集

`/files/{fileId}` を開くとHocuspocusへ接続し、Excalidraw要素をYjs documentへ同期する。 image挿入、Note embeddable、appState復元にも対応する。

Agent run

Run buttonやauto modeで依頼

promptは `agentRunRequests` に入り、serverがworkerへ渡す。workerはCodexで estimate、draft、final artifactを作る。

Review

提案はghostとして出る

Agentの成果物は半透明のghost elementとしてcanvasに追加される。 人間がApproveすると通常要素へ昇格し、Rejectすると削除扱いになる。

内部構成

3つのアプリと4つのpackagesが、Yjs documentを中心に役割分担している。

apps/web

Browser UI

Excalidraw 人間の描画、画像挿入、Note表示、footer tools。
HocuspocusProvider Yjs documentとawarenessを同期する。
Proposal controls 最新proposalをapprove/rejectする。
apps/server

Coordination

Hono API file作成、import、status、agent run queue。
Hocuspocus Yjs documentをload/storeし、変更時にrequestを検出。
AgentSupervisor fileごとのworker processを起動・監視する。
apps/worker

Agent Runtime

Workspace `~/.excalidraw-agent/{fileId}` にrun artifactを保存。
Codex SDK estimate、draft、final.excalidrawを作成。
Proposal pipeline baseとの差分からghost proposalをpublish。
shared packages: IDs, metadata, Yjs element helpers, proposal helpers

Agent run の中身

workerは一気に最終結果だけを出すのではなく、見積もり、draft、品質チェックを段階的に行う。

queued → running

`agentRunRequests` から1件取り出し、`agentRuns[runId]` を作る。

snapshot

現在のcanvas、assets、notesを `base-scene.json` として保存し、base revisionを計算する。

estimate

Codexが必要領域と作業stepsを `estimate.json` に書く。workerが衝突しにくいplanned areaへ調整する。

drafting

stepごとに `draft-step-n.excalidraw` を作り、途中結果をdraft ghostとして表示できる。

verifying

`final.excalidraw` を読み、構造、要素数、planned areaとの重なりを確認する。失敗時は1回だけ修正turnを行う。

proposed

baseとfinalの差分からadd proposalを作り、半透明のghost elementとしてYjsへpublishする。

今の制約

Apply

反映はadd中心

update、delete、moveはpatchとして検出されるが、approve時はconflict/stale側へ倒す。

Model

UIのmodel選択は未接続

Quality
